Bienvenid@,
  • ¿Por qué es importante hacer un session_unset() y luego un session_destroy()?
    Porque session_unset() sólo libera las variables de la sesión, es decir, la sesión aún existe en la computadora del usuario. Por lo tanto, es necesario hacer un session_destroy() para destruir todos los datos asociados a la sesión.
  • ¿Cuál es la diferencia entre una variable de sesión y una cookie?
    Una cookie es un fragmento de información que el navegador web almacena en la computadora del usuario, mientras que una variable de sesión se almacena en el servidor, jamás en el cliente.
  • ¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?
    Se le asigna un ID a la fotografía, ya sea uno creado de forma automática, o uno compuesto por un timestamp.
  • ¿Qué es CSRF y cómo puede prevenirse?
    CSRF significa Cross-Site Request Forgery, es un ataque que fuerza al navegador web de su víctima a enviar una petición a una aplicación web vulnerable en la que había iniciado sesión. Como usuario, puede prevenirse cerrando sesiones cuando hemos dejado de usarlas. Como desarrolladores, se pueden prevenir asociando al usuario un tipo de id único llamado token, que debe verificarse después para comprobar la identidad del usuario.
