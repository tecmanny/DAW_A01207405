Que es Material Design?

Es una guia de estilo que Google creo hace aproximadamente 4 años con el objetivo de unificar criterios de estilos funcionales y esteticos

Basicamente la creacion de interfaz para sistemas operativos

Material Design nace de la necesidad de lograr una buena coherencia en diseños de productos digitales y experiencias de usuario. Se crea en base a que el diseño es el arte de la resolución de problemas continuos en el que se lleva a cabo un proceso de investigación, elaboración y desarrollo de ideas.

Investigaciones de LAB 1

¿Cuál es la diferencia entre Internet y la WWW?
- La internet es la infraestructura física sobre la que se ejecuta la World Wide Web

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET es un método de HTML que se utiliza para pedir datos de una fuente específica
POST envía datos para que sean procesados por una fuente específica
HEAD es similar a GET pero regresa solo headers de HTML pero no el cuerpo de un documento
PUT sube o manda un representación del identificador URI específico
PATCH es utilizado para subir recursos parciales, cuando sólo se quiere actualizar una parte
DELETE borra el recurso especificado

¿Qué método HTTP se debe utilizar al enviar un formulario HTML como el de la encuesta que constestaste al principio de la actividad? ¿Por qué?
- Probablemente POST ya que lo que se quiere hacer es mandar información que será procesada luego.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
- Se utiliza el método GET ya que se quiere acceder a un recurso específico.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
- Indica que el request fue procesado correctamente, normalmente este código no es visible.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
- Posiblemente ya que marca que no se encuentra el recurso en ese servidor. Podría ser que el desarrollador no subió correctamente los recursos. De otro modo
puede se que el usuario intentara acceder a un recurso inexistente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
- No generalmente ya que es un error interno del servidor y generalmente concierne más a un sysadmin del sistema.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
- Significa que son atributos que han sido reemplazados por alternativas más flexibles o funcionales, como HTML o CSS, generalmente los buscadores aun les dan
soporte, sin embargo son obsoletas y su soporte no puede ser garantizado.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
- Generalmente la diferencia más grande entre las diferentes versiones de HTML es que HTML5 dejará de dar soporte a muchas opciones de layout de página que
han sido usados erroneamente por los desarrolladores. Se pretende que para el layout en HTML5 se utlice mucho más CSS. Así también HTML5 tiene ahora soporte
para muchas tecnologías nuevas como audio y vídeo integrados ya en el lenguaje.

¿Qué componentes de estructura y estilo tiene una tabla?
- Las tablas tienen celdas ordenadas en columnas y filas así como también estilos de borde, "padding", "headings", espacio entre bordes y captions.

¿Cuáles son los principales controles de una forma HTML5?
- Class, style, id, method, action, type, name, value, disabled, readonly, tabindex y acceskey.

¿Qué tanto soporte HTML5 tiene el navegador que utilizas?
- Mi navegador, Chrome 47 de 64b, obtiene 521 de 555 puntos de soporte en HTML5
